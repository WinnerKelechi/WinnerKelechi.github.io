FAMILY-CENTRIC SOCIAL NETWORK SYSTEM

Position: Lead Developer
Stack: PHP, JavaScript, MySQL, Bootstrap, AJAX, JASON and HTML/CSS

The proposed system is a web based family centric social network system. It is a system used to create family trees and communication is done through chats. It provides details of each members of a family tree and their relationships. The proposed system creates a family tree based on the relationship of the members with the person who started the tree.
The member of a family can then add other members to the tree, by choosing their parents. Unlike the existing systems, the proposed system contains a platform where members of the family can communicate, hence the social network.
The system has two actors, which includes:
a) System administrator: The system admin is the member of the family that starts the tree.
b) Members: Members of a family tree are those related to the admin by blood or marriage.

Functional Requirement
i. A member of the family, who would act as the system’s admin must register.
ii. Admin will receive family identification number upon completion of registration.
iii. Admin will be able to add a new member and their relationship. iv. Members will be able to access the family tree with a unique identification number.
v. Members will be able to update their information.
vi. Add member function to enable the system create the proper tree structure.
vii. A Chat platform: Where members the family can communicate.
viii. The system also will be able to give a breakdown of the family’s statistics, showing the percentage of male to female, percentage of members born in a particular city and a scale of birth years (from oldest to youngest).
ix. System will be able to suggest a family member based on the family name/surname and village or state of origin.
x. Members will be able to update profile.

CREDIT CARD FRAUD DETECTION ON AN E-STORE

Position: Lead Developer
Stack: PHP, JavaScript, MySQL, Bootstrap, AJAX, JASON and HTML/CSS

This project shows that credit card fraud can be detected using Deep Learning Algorithm during transactions. Deep Learning Algorithm helps to obtain a high fraud detection coverage combined with a low false alarm rate.
The following are the advantages of proposed system over the existing system:
a. The detection of fraudulent use of the card is found much faster than the existing system.
b. The System can send notification to the admin that a fraudulent transaction have been detected.
c. The system admin can easily block the card when he/she finds that the card is being stolen.
d. We can find the most accurate detection using this technique.
e. Maximizes the correct prediction and maintains the incorrect prediction at an acceptable level.
f. The credit card fraud detection system shall not cause physical harm to users and nonusers.
g. The credit card fraud detection system will not cause interference to external systems.

Functional Requirement
i. This system is a web-based system that allows card users to register by providing card details and personal information.
ii. The system will be able to detect fraudulent transactions based on the card owner’s activities.
iii. The system will throw security questions once a transaction is suspected to be fraudulent.
iv. The system rejects transaction if security checks are not met. v. The users will have a user friendly interface.

First, the consumer registers and provides a second level security check, he/she can then login and order the product via internet by using the credit card payment method. After that, the system saves the amount of money, date and time of payment and location.

Now, this is the credit card fraud detection system used to validate the behavior of credit card. Each time a transaction is made, the credit card fraud system requests the consumer’s profile from the database to bring his behavior into the AE and RBM by using deep learning. Based on the AE, the transaction proceeds to the second level security check. Then, the AE uses past behaviors to be trained first, and then uses the new coming transaction as a validation test for the transaction. AE does not use labeled transactions to be trained, because it is unsupervised learning. RBM uses all transactions as visible input and then that goes to the hidden node, and after the calculation of the activation function, the RBM reconstructs the model by transferring the new input from the activation function back to the output or visible function. If the transaction is fraudulent, the system will record this transaction as a fraud in the database and will then reject it.
Next, the system administrator will receive a notification to block the card temporarily and send an SMS alert to the real consumer that the transaction has not been processed, because the system suspects the transaction as fraudulent.

EXAMINATION AND COURSE REGISTRATION SYSTEM

Position: Lead Developer
Stack: PHP, JavaScript, MySQL, Bootstrap, AJAX, JASON and HTML/CSS

This system is designed to solve problems associated course registration, scheduling and writing exams in universities. There are to part the system design processes namely: Administrators and Students.

Administrator
The system administrator is one that has fully access to all the system functionalities. In this project, the system administrators are the Examination officer, Head of Department and Lecturers. The Examination officer plays the role of the super-admin, as he/she is saddled with the responsibility of adding other lecturers to the system. This is done to prevent imposters from gaining access to the admin side of the system. Once a lecturer is added to the system by the admin, he will receive a token generated dynamically by the system. This token will serve as the lecturer’s ID at the point of students’ registration. No two lecturers can have the same ID. The super-admin (Examination officer) will provide the lecturer with a username and a password which he will use to log in to the system through the admin login form.

Students
The system gives room for students to remotely register for an exam and write the exam without the constraint of a physical office or venue. To gain access to the system, student must first register by clicking on “Student’s Registration” on the index page, then click on the register button, which opens to a registration form.

A TIMETABLE MANAGEMENT SYSTEM FOR UNIVERSITY OF CALABAR COMPUTER SCIENCE DEPARTMENT

Position: Lead Developer
Stack: PHP, JavaScript, MySQL, Bootstrap, AJAX, JASON and HTML/CSS

This system was developed to solve the exam timetabling problem being faced by universities every academic year and reduce high cost and slow turnaround involved in the generation of near-optimal timetables. The system has capabilities for input of the various courses, halls of examinations, departments, programs, buildings, lecturers and the specification of a few constraints from which the timetable is constructed. The proposed timetabling system for this project seeks to generate near optimal timetables using the principles of genetic algorithm (selection and crossover).

ADVANTAGES OF THIS SYSTEM
The timetable generation process by the education center staff is:
a. Unlike the manual timetabling system, the system offers flexibility.
b. It utilizes minimal processing/computing power.
c. It greatly reduces the time needed to generate near-optimal timetables.
d. It provides an easy means for data entry and revision through an intuitive interface.
e. It increases productivity.
f. It almost eliminates paperwork.
g. It simplifies the timetabling process.

Functional Requirement
i. The timetable officer who will act as the system’s administrator should be able to login to the admin-panel.
ii. Admin should be able to update and add new halls.
iii. Admin should also add courses and lecturers.
iv. The system should be able to generate timetable without clashes of time and venue.
v. Students should be able to view the generated timetable.
